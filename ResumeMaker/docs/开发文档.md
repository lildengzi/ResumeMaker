# 智能简历生成器 - 详细开发文档

> 版本：1.0  
> 基于现有 Streamlit + LangChain 代码扩展  
> 目标：实现可手动微调、多智能体、多模板、JD多源输入的智能简历生成器

---

## 1. 项目概述

### 1.1 背景
用户已实现一个初步的智能简历生成器，包含：
- 基础个人信息、教育、技能、项目等表单编辑；
- Streamlit 界面，支持模板切换与实时预览；
- LangChain Agent（信息收集、简历改写）基于 JD 做优化；
- PDF 导出。

但当前版本在 **前端交互灵活性**、**JD 多源输入**、**智能体架构扩展性** 以及 **手动微调能力** 上存在不足。

### 1.2 目标
构建一个 **高交互、可扩展** 的智能简历生成器，实现：
- 左侧配置栏（JD 输入、样式微调、个人信息上传）+ 右侧实时简历预览与手动编辑；
- 多种美观模板 + 动态调节字体、边距、行高等排版参数；
- JD 输入：手写文字、OCR 图片识别、网页链接抓取（可选）；
- 简历生成后可逐项微调（所见即所得）；
- 后端采用 **Agent 工厂模式 + 工作流编排**，预留简历打分、面试辅助等智能体；
- 工具调用严格遵守边界，禁止智能体越权；
- 配置文件统一管理 LLM 密钥与参数。

class BaseResumeAgent(ABC):

    @abstractmethod

        pass

        # 返回收集的事实字典
        pass

class ResumeWriterAgent(BaseResumeAgent):
    async def run(self, state: dict) -> dict:
        # 基于 state["collected_facts"] 和 state["current_resume"]
        # 调用 LLM 生成新的 resume_data
        pass

# 工厂
class AgentFactory:
    _agents = {
        "info_collector": InfoCollectorAgent,
        "resume_writer": ResumeWriterAgent,
        # "resume_scorer": ResumeScorerAgent,  # 待实现
    }
    @classmethod
    def create(cls, agent_type: str, llm: ChatOpenAI, tools: List[BaseTool]):
        return cls._agents[agent_type](llm, tools)

#### 4.2.3 工作流编排层（LangGraph）
定义工作流状态：
```python
class WorkflowState(TypedDict):
    jd_text: str
    style_params: dict
    uploaded_files: List[dict]   # 路径、类型、内容摘要
    collected_facts: dict
    current_resume: dict
    optimized_resume: dict
    final_markdown: str
    error: Optional[str]
```

构建图：
```python
from langgraph.graph import StateGraph, END

workflow = StateGraph(WorkflowState)
workflow.add_node("collect_info", collect_info_node)
workflow.add_node("write_resume", write_resume_node)
workflow.add_edge("collect_info", "write_resume")
workflow.add_edge("write_resume", END)
workflow.set_entry_point("collect_info")
app = workflow.compile()
```

> **注意**：当前 `smart_resume_core.py` 中 `SmartResumeOrchestrator` 是顺序调用，可重构为基于 LangGraph 的图，便于未来加入条件分支（如质量评分不合格则重新收集信息）。

#### 4.2.4 工具层设计（禁止逃逸）
工具必须运行在受限环境：
- 文件读写仅限工作空间目录（`workspace`）。
- 网络请求限制域名（白名单）或只允许特定 API（如 OCR 服务）。
- 所有工具返回结果需经过 `extract_json_block` 清洗，避免注入。
- 为每个工具增加权限标识（如 `@tool(permission="read_file")`），在调用前由中间件检查。

#### 4.2.5 简历生成工具：明确回答
用户问：简历生成工具是作为智能体工具还是直接实现？
**决策**：**不作为工具，而是由 `ResumeWriterAgent` 直接调用 LLM 生成 JSON**。  
理由：
- 简历生成是 Agent 的核心职责，不应剥离成独立工具（工具应当是通用、可复用的原子能力）。
- 工具调用会增加一次 LLM 调用开销，且 Agent 需要额外解析工具结果，增加复杂度。
- 保持当前设计：`ResumeWriterAgent` 直接输出 `resume_data` JSON。

> 但是，可以将“渲染 Markdown/PDF”作为工具提供给 Agent 用于预览调试，但非核心。

## 6. 接口设计（供前后端分离时参考）

若未来将后端抽离为独立 API，设计如下 RESTful 接口：

| 端点 | 方法 | 功能 | 请求体 |
|------|------|------|--------|
| `/api/jd/ocr` | POST | OCR 识别图片 | `image: file` |
| `/api/jd/fetch` | POST | 网页抓取 JD | `url: string` |
| `/api/files/upload` | POST | 上传头像/简历/README | `file: file`, `type: string` |
| `/api/resume/generate` | POST | 触发智能生成 | `{ "jd_text": "...", "style_params": {...}, "file_ids": [...] }` |
| `/api/resume/update` | PUT | 手动修改某字段 | `{ "section": "skills", "index": 0, "value": "..." }` |
| `/api/resume/export` | POST | 导出 PDF | `{ "resume_data": {...}, "style_params": {...} }` |

Streamlit 前端可通过 `requests` 调用这些接口（或仍直接调用 Python 函数）。

## 7. 配置管理示例

```python
# config.py
from pydantic import BaseSettings, Field

class LLMConfig(BaseSettings):
    provider: str = "openai"
    model: str = "gpt-4o-mini"
    api_key: str = Field(..., env="OPENAI_API_KEY")
    base_url: Optional[str] = None
    temperature: float = 0.2

class ToolConfig(BaseSettings):
    ocr_engine: str = "paddleocr"
    web_timeout: int = 10

class Settings(BaseSettings):
    llm: LLMConfig = LLMConfig()
    tools: ToolConfig = ToolConfig()
    workspace_dir: str = "./workspace"
    max_file_size_mb: int = 10

    class Config:
        env_file = ".env"
        env_nested_delimiter = "__"

settings = Settings()
```

## 8. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| LLM 输出不符合 JSON 结构 | 生成失败 | 增加 JSON 修复 LLM 调用作为 fallback；前端捕获异常提示用户重试 |
| OCR 识别准确率低 | JD 信息缺失 | 提供手动修改 OCR 结果的文本框 |
| 网页抓取被反爬 | 无法获取 JD | 优先使用缓存，或提示用户手动输入 |
| 动态样式微调导致 PDF 排版错位 | 用户体验下降 | 提供“重置默认样式”按钮；PDF 生成时强制适配 A4 尺寸 |
| 多智能体调用链超时 | 长时间无响应 | 使用异步任务队列（Celery），前端轮询结果 |
