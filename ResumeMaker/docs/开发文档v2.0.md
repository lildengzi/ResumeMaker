# 智能简历生成器 - 最终开发文档（一站式定制方案）

> 版本：4.0  
> 更新日期：2026-05-11  
> 产品定位：**从快速生成到高度定制的一站式简历解决方案**，支持用户随意增删模块、调整样式、借助智能体优化内容。

---

## 1. 核心设计原则

- **模块化**：简历由多个**独立模块**组成，每个模块有标题、内容（支持文本/列表/日期等）、可见性、排序。
- **可扩展**：用户可以自由添加/删除/重命名模块，无需修改代码。
- **样式全可调**：主题色、字体、边距、行高、背景色等均可由用户调节。
- **智能体可选**：用户可选择是否启用 AI 优化，并可输入自然语言建议指导 AI。
- **输出美观**：HTML 预览 + PDF 导出风格统一，支持多模板。

---

## 2. 输入输出定义

### 2.1 输入（用户提供）

| 输入项 | 必选/可选 | 说明 |
|--------|-----------|------|
| **个人信息** | 必选 | 姓名、应聘岗位、城市、电话、邮箱、GitHub、照片等（基础字段） |
| **项目经历** | 可选 | 可手动添加，也可通过上传 README 或网站抓取自动生成 |
| **网站抓取** | 可选 | 输入项目或个人网站 URL，自动提取关键信息（作为项目经历补充） |
| **岗位描述 (JD)** | 可选 | 粘贴 JD 文本，用于智能体优化 |
| **微调建议** | 可选 | 自然语言指导，如“突出量化的成果” |

### 2.2 输出

- **美化后的简历**：HTML 实时预览 + PDF 下载 + Markdown 导出。
- **可微调项**：
  - 简历模块：增删改模块、重排顺序、修改标题。
  - 样式：主题色（主色、辅助色）、全局字体大小、行高、页面边距、模块间距。
  - 内容：每个模块的内容直接编辑（文本、列表）。

## 4. 功能模块划分

### 4.1 输入处理
- **个人信息表单**：基础字段编辑。
- **项目经历导入**：
  - 手动添加项目模块（标题、时间、描述、技术栈）。
  - 上传 README.md：解析文本，生成项目模块草稿。
  - 输入 URL：调用 `tools/web_tool.py` 抓取页面内容，提取项目描述（需接入真实抓取库）。
- **JD 输入**：文本框或文件上传（OCR 预留）。
- **微调建议**：文本框，用于指导 AI。

### 4.2 模块管理（左侧 Sidebar）
- 显示所有模块列表，支持：
  - 添加新模块（选择类型：教育/技能/项目/工作经历/自定义）。
  - 编辑模块标题、内容。
  - 删除模块。
  - 拖拽或按钮上下移动模块顺序。
  - 切换可见性（隐藏/显示）。
- 每个模块的内容编辑器根据 `type` 提供不同的表单（如教育经历有学校、学位等字段；自定义模块则为简单的标题+文本/列表）。

### 4.3 样式定制（左侧 Sidebar）
- 主题色选择器（`st.color_picker`）。
- 滑块：字号（10-24px）、边距（10-80px）、行高（1.0-2.5）、模块间距（8-40px）。
- 复选框：显示/隐藏头像。
- 即时生效：修改后自动刷新右侧预览。

### 4.4 智能体工作流
- **可选启用**：点击“智能优化”前，用户可选择是否使用 AI。
- **输入**：当前简历 JSON（或精简摘要）、JD 文本、用户微调建议。
- **处理**：
  1. 调用 `ResumeWriterAgent`，生成优化后的 `modules` 内容（只修改内容，不改变模块结构）。
  2. （可选）调用 `ResumeJudgeAgent`，返回质量评估。
- **输出**：更新 `resume_data` 中的 `modules`，预览刷新。

### 4.5 渲染与导出
- **HTML 预览**：根据 `modules` 和 `style` 动态生成 A4 风格页面（`renderers/html_renderer.py`）。
- **PDF 导出**：基于 HTML 转 PDF（推荐 Playwright）或保持 ReportLab 简单实现。
- **Markdown 导出**：将模块内容转为 Markdown 格式。

## 5. 关键技术实现要点

### 5.1 动态模块编辑器（`ui/module_editor.py`）

每个模块独立渲染编辑器，使用 Streamlit 的 `form` 或 `expander`，支持动态增删内容项（如教育经历可添加多条）。

例如教育模块的编辑器：

```python
def render_education_editor(module):
    items = module["content"].get("items", [])
    for idx, edu in enumerate(items):
        with st.container():
            col1, col2 = st.columns([3,1])
            edu["school"] = col1.text_input("学校", edu.get("school",""), key=f"edu_school_{idx}")
            if col2.button("删除", key=f"del_edu_{idx}"):
                items.pop(idx)
                st.rerun()
            edu["degree"] = st.text_input("学位", edu.get("degree",""), key=f"edu_degree_{idx}")
            edu["major"] = st.text_input("专业", edu.get("major",""), key=f"edu_major_{idx}")
            edu["startDate"] = st.text_input("开始时间", edu.get("startDate",""), key=f"edu_start_{idx}")
            edu["endDate"] = st.text_input("结束时间", edu.get("endDate",""), key=f"edu_end_{idx}")
            edu["description"] = st.text_area("描述", edu.get("description",""), key=f"edu_desc_{idx}")
    if st.button("+ 添加教育经历"):
        items.append({"school":"","degree":"","major":"","startDate":"","endDate":"","description":""})
        st.rerun()
```

对于自定义模块，提供一个通用的键值对编辑器：

```python
def render_custom_editor(module):
    items = module["content"].get("items", [])
    for idx, item in enumerate(items):
        # 简单起见，每个自定义项为两个字段：title 和 description
        item["title"] = st.text_input("标题", item.get("title",""), key=f"custom_title_{idx}")
        item["description"] = st.text_area("内容", item.get("description",""), key=f"custom_desc_{idx}")
        if st.button("删除", key=f"del_custom_{idx}"):
            items.pop(idx)
            st.rerun()
    if st.button("+ 添加条目"):
        items.append({"title":"","description":""})
        st.rerun()
```

### 5.2 样式与预览同步

- 样式参数存储在 `st.session_state.style_params`，通过 `on_change` 回调或每次滑块变动时自动调用 `refresh_preview()`。
- 预览 HTML 生成函数 `build_full_preview_document(resume_data, style_params)` 使用最新参数渲染。

### 5.3 智能体摘要生成

为避免过于复杂的 JSON 传递，摘要生成函数如下：

```python
def generate_resume_summary(resume_data):
    lines = [f"姓名：{resume_data['basics']['name']}，应聘：{resume_data['basics']['headline']}"]
    for mod in resume_data["modules"]:
        lines.append(f"\n## {mod['title']}")
        if mod["type"] == "skills":
            lines.append(f"技能：{', '.join(mod['content'].get('items', []))}")
        elif mod["type"] == "education":
            for edu in mod["content"].get("items", []):
                lines.append(f"- {edu.get('degree')}，{edu.get('school')}，{edu.get('major')}，{edu.get('startDate')}~{edu.get('endDate')}：{edu.get('description')}")
        else:  # custom 及其他
            for item in mod["content"].get("items", []):
                if isinstance(item, dict):
                    lines.append(f"{item.get('title', '')}: {item.get('description', '')}")
                else:
                    lines.append(f"- {item}")
    return "\n".join(lines)
```

智能体收到摘要 + JD + 用户建议 → 返回修改后的内容（通常是针对某些模块的文本建议）。然后系统将建议应用到对应的模块内容中（简单替换）。

### 5.4 性能优化

- 使用 `@st.cache_resource` 缓存 LLM 实例。
- 预览 HTML 生成结果缓存（依赖 `resume_data` 和 `style_params` 的哈希值），仅在变化时重新生成。
- 首次加载时显示骨架屏（简单文字提示），后台异步加载无大影响。

def migrate_legacy_resume(old_data):

## 9. 最终产品体验预期

- 用户打开网页，看到左侧配置区、右侧空白预览。
- 快速填写姓名、岗位，选择默认模块（教育、技能），点击“智能生成”获得一份基础简历。
- 用户不满意模块：可以添加“获奖经历”模块，自由填写。
- 用户不满意样式：拖动滑块改变主题色、字体大小，预览实时变化。
- 用户希望突出某些内容：在微调建议框写“请强调我的创业经历”，智能体重新生成内容。
- 导出 PDF 获得与 HTML 预览完全一致的文档。
